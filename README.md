### 💡 Проблема и Решение

| Проблема | Реализованное решение |
| :--- | :--- |
| Неудобная подготовка по статичным `.docx` файлам без интерактива и поиска. | Создано легкое SPA, работающее по прямой ссылке. Никакой установки, бэкенда или регистрации. |
| Хаотичное зазубривание без системы запоминания. | Внедрен алгоритм **интервального повторения**: ошибочные ответы автоматически сохраняются и предлагаются снова до полного усвоения. |
| Необходимость делить тесты между разными учебными группами. | Реализована **динамическая загрузка** файлов `.json` напрямую из разных веток (branches) GitHub-репозитория через GitHub REST API. |

---

### 🚀 Ключевые возможности

- 🧠 **4 режима работы:** 
  1. *Обычный* (все вопросы подряд)
  2. *Тест-кабинет* (50 случайных вопросов с ограничением)
  3. *Быстрое повторение* (с пагинацией по 20 вопросов и живым поиском)
  4. *Выбор диапазона* (ручной выбор конкретных номеров вопросов)
- 💾 **Умное сохранение состояния:** Весь прогресс (текущий вопрос, таймер, ошибки, очки) сохраняется в `localStorage`. Можно закрыть вкладку и продолжить с того же места в течение 48 часов.
- 🌓 **Темная и светлая тема:** Самописная CSS-реализация с запоминанием выбора пользователя.
- 📐 **Рендеринг формул:** Полная интеграция `MathJax` для корректного и красивого отображения сложных математических выражений.
- 📱 **Полная адаптивность:** Удобный интерфейс как для десктопа, так и для мобильных устройств.

### 🛠️ Архитектура и Стек

Проект создан с упором на производительность и отсутствие зависимостей от серверной части (Serverless):
- **Core:** Vanilla JavaScript (ES6+), HTML5, CSS3 (без тяжелых фреймворков).
- **Data:** Динамическая загрузка через `GitHub REST API` + локальный `FileReader API` для загрузки пользовательских `.json` файлов прямо в браузере.
- **Storage:** `localStorage` для управления состоянием приложения (`testAppState`) и хранения месячной статистики результатов.
